**엔드포인트 3개 (URL 패턴 고정):**

1. GET `/scenarios/{scenario_id}/variants/{variant_id}/graph`
   - response_model=CanonicalScenarioGraph
   - Depends: db=Depends(get_db)
   - 구현: get_canonical_graph(db, scenario_id, variant_id) → None이면 raise NoResultFound(f"scenario '{scenario_id}' / variant '{variant_id}' not found") → 아니면 graph 반환

2. GET `/scenarios/{scenario_id}/variants/{variant_id}/resolve`
   - response_model=ResolverResult
   - Depends: db=Depends(get_db)
   - 구현: graph = get_canonical_graph(db, scenario_id, variant_id) → None이면 raise NoResultFound(...) → resolve(graph) 반환
   - RuleCache 의존성 불필요 (Resolver는 gate_rules 무관 — D-05 참조)

3. GET `/scenarios/{scenario_id}/variants/{variant_id}/gate`
   - response_model=GateExecutionResult
   - Depends: db=Depends(get_db), cache: RuleCache=Depends(get_rule_cache)
   - 구현: graph = get_canonical_graph(db, scenario_id, variant_id) → None이면 raise NoResultFound(...) → evaluate_gate(graph, cache.gate_rules) 반환
   - D-05: cache.gate_rules가 빈 리스트여도 503 없이 evaluate_gate(graph, []) 호출 → status=PASS
