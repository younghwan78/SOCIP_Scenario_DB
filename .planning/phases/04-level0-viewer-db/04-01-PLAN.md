### Task 1: SwStackNode Pydantic 모델 추가 + Pipeline.sw_stack 필드

**File**: `src/scenario_db/models/definition/usecase.py`
**Type**: modify
**Depends on**: none

`Pipeline` 모델은 `BaseScenarioModel`을 상속하므로 `extra='forbid'`가 적용되어 있다. `sw_stack` YAML 섹션을 추가하면 ETL 시 ValidationError가 발생한다. 이를 방지하려면 `SwStackNode` 모델과 `Pipeline.sw_stack` 필드를 먼저 추가해야 한다.

구현 내용:

1. `Pipeline` 클래스 정의 앞에 `SwStackNode` 클래스를 추가한다:

```python
class SwStackNode(BaseScenarioModel):
    layer: Literal["app", "framework", "hal", "kernel"]
    id: str
    label: str
    ip_ref: str | None = None  # 연결 HW IP pipeline node id (e.g., "csis0", "mfc")
```

2. `Pipeline` 클래스에 `sw_stack` 필드를 추가한다 (기존 `edges` 필드 바로 뒤):

```python
sw_stack: list[SwStackNode] = Field(default_factory=list)
```

3. `Pipeline._validate_edge_references` model_validator는 그대로 유지 — sw_stack의 `ip_ref`는 HW pipeline node id를 가리키지만, 교차 검증은 Phase 4 scope 밖. validator 수정 없음.

4. `usecase.py` 최상단 import에 `SwStackNode`를 위한 `Literal` import가 이미 있는지 확인 (`from typing import Literal` 존재). 없으면 추가.

주의사항:
- `BaseScenarioModel` 상속으로 `model_config = ConfigDict(extra='forbid')`가 이미 적용됨 — 별도 config 선언 불필요
- `ip_ref` 필드는 YAML의 `ip_ref: csis0` 값을 받음 — pipeline node id이지 ip catalog id가 아님 (e.g., "csis0" not "ip-csis-v8")
- `SwStackNode.layer`의 `Literal` 타입이 `"hw"` 레이어를 포함하지 않도록 주의 — SW 레이어만 (`app/framework/hal/kernel`)

Key implementation notes:
- `usecase.py` 파일은 반드시 먼저 읽고 수정할 것 (파일 전체 구조 파악 필수)
- `SwStackNode` 정의 위치: `Pipeline` 클래스 정의 바로 앞 (약 45번 라인 부근)
- `Pipeline` 모델의 `_validate_edge_references`는 `self.nodes`와 `self.edges`만 검사 — sw_stack 영향 없음

Acceptance criteria:
- [ ] `SwStackNode` 클래스가 `usecase.py`에 존재하며 `layer/id/label/ip_ref` 필드를 가진다
- [ ] `Pipeline.sw_stack: list[SwStackNode] = Field(default_factory=list)` 필드가 존재한다

### Task 3: service.py — CATEGORY_TO_LANE + topological sort + 실좌표 계산

**File**: `src/scenario_db/view/service.py`
**Type**: modify
**Depends on**: none (Task 1과 병렬 가능)

`_projection_to_view_response()` 함수를 확장하여 실좌표를 계산한다. 현재 stub의 `position={"x": 0.0, "y": 0.0}` 코드를 실좌표로 교체한다.

**구현 세부 내용:**

파일 상단 import에 추가:

```python
import graphlib
```

`CATEGORY_TO_LANE` 상수를 모듈 레벨에 정의 (함수 밖):

```python
CATEGORY_TO_LANE: dict[str, str] = {
    "camera":  "hw",   # CSIS, ISP — ip-csis-v8.yaml, ip-isp-v12.yaml에서 확인
    "codec":   "hw",   # MFC — ip-mfc-v14.yaml에서 확인
    "display": "hw",   # DPU — ip-dpu-v9.yaml에서 확인
    "memory":  "hw",   # LLC — ip-llc-v2.yaml에서 확인
}

STAGE_STEP: int = 310  # capture→processing 간격 (layout.py STAGE_X 기반)
```

`_projection_to_view_response(projection: dict) -> ViewResponse` 함수 전체를 다음 로직으로 교체:

1. **IP catalog 조회**: `projection.get("ip_catalog", [])` → `{ip_id: ip_dict}` 딕셔너리 구성

2. **topological sort (graphlib)**:
   ```python
   ts = graphlib.TopologicalSorter()
   node_ids = {n["id"] for n in nodes_raw}
   for n in nodes_raw:
       ts.add(n["id"])
   for e in edges_raw:
       src, tgt = e.get("from"), e.get("to")
       if src in node_ids and tgt in node_ids:
           ts.add(tgt, src)  # tgt depends on src
   ts.prepare()
   stage_map: dict[str, int] = {}
   stage_idx = 0
   while ts.is_active():
       batch = list(ts.get_ready())
       for nid in batch:
           stage_map[nid] = stage_idx
       ts.done(*batch)
       stage_idx += 1
   ```

3. **노드 좌표 계산**:
   ```python
   for node in nodes_raw:
       nid = node["id"]
       ip_ref = node.get("ip_ref", "")
       ip_info = ip_catalog.get(ip_ref, {})
       category = ip_info.get("category", "")
       lane = CATEGORY_TO_LANE.get(category, "hw")
       s_idx = stage_map.get(nid, 0)
       x = float(LANE_LABEL_W + s_idx * STAGE_STEP + STAGE_STEP // 2)
       y = LANE_Y[lane]
       nodes.append(NodeElement(
           data=NodeData(id=nid, label=node.get("label", nid), type="ip", layer=lane),
           position={"x": x, "y": y},
       ))
   ```

4. **엣지 변환**: pipeline edges를 EdgeElement로 변환 (`from_`→`from` alias 주의):
   ```python
   for e in edges_raw:
       src = e.get("from")
       tgt = e.get("to")
       etype = e.get("type", "M2M")
       # EdgeData.flow_type Literal: "OTF", "vOTF", "M2M", "control", "risk"
       # PipelineEdge.type: "OTF", "M2M" (EdgeType StrEnum)
       flow_type = etype if etype in {"OTF", "vOTF", "M2M", "control", "risk"} else "M2M"
       edge_id = f"e-{src}-{tgt}"
       edges.append(EdgeElement(data=EdgeData(
           id=edge_id, source=src, target=tgt, flow_type=flow_type
       )))
   ```

5. **ViewSummary** 구성 — 기존 stub과 동일하게 유지 (period_ms=0.0 등 placeholder 값은 Wave 2에서 보완 가능)

6. **`project_level0()` 함수**: topology 분기의 `raise NotImplementedError("topology mode is Phase 4 work")` 주석만 업데이트하되 실제 구현은 Wave 2 (04-PLAN-03)에서 수행. 현재는 `raise NotImplementedError("topology mode: implemented in 04-PLAN-03")`로 메시지만 변경.

Key implementation notes:
- `ip_catalog` 데이터가 `projection` dict에 포함되는지 확인 필요. `get_view_projection()` 반환 구조가 `{"scenario_id": ..., "variant_id": ..., "pipeline": {...}, "ip_catalog": [...]}` 형태인지 service.py를 읽기 전에 `src/scenario_db/db/repositories/view_projection.py`를 먼저 확인할 것
- `LANE_Y`는 `from scenario_db.view.layout import LANE_Y, LANE_LABEL_W` — 이미 import되어 있음
- `graphlib.TopologicalSorter`는 Python 3.9+ 표준 라이브러리 — `static_order_groups()` 메서드는 존재하지 않음. `prepare() + is_active() + get_ready() + done()` 패턴만 사용
- `PipelineEdge`의 `from_` 필드(alias="from") — projection dict에서는 `"from"` 키로 존재하므로 `e.get("from")` 사용

Acceptance criteria:

- [ ] CATEGORY_TO_LANE 상수가 모듈 레벨에 정의되어 있다

import pytest
from scenario_db.view.service import _projection_to_view_response, CATEGORY_TO_LANE


SAMPLE_PROJECTION = {
    "scenario_id": "uc-test",
    "variant_id":  "v-test",
    "project_name": "Test Project",
    "pipeline": {
        "nodes": [
            {"id": "csis0", "ip_ref": "ip-csis-v8"},
            {"id": "isp0",  "ip_ref": "ip-isp-v12"},
            {"id": "mfc",   "ip_ref": "ip-mfc-v14"},
        ],
        "edges": [
            {"from": "csis0", "to": "isp0", "type": "OTF"},
            {"from": "isp0",  "to": "mfc",  "type": "M2M"},
        ],
    },
    "ip_catalog": [
        {"id": "ip-csis-v8",  "category": "camera"},
        {"id": "ip-isp-v12",  "category": "camera"},
        {"id": "ip-mfc-v14",  "category": "codec"},
    ],
}


def test_all_nodes_assigned_hw_lane():
    """camera/codec category → hw lane."""
    view = _projection_to_view_response(SAMPLE_PROJECTION)
    for node in view.nodes:
        assert node.data.layer == "hw", f"Node {node.data.id} has layer {node.data.layer}, expected hw"


def test_x_coordinates_increase_with_topology():
    """topological sort → csis0(0) → isp0(1) → mfc(2); x좌표 단조 증가."""
    view = _projection_to_view_response(SAMPLE_PROJECTION)
    pos = {n.data.id: n.position["x"] for n in view.nodes}
    assert pos["csis0"] < pos["isp0"] < pos["mfc"], (
        f"Expected csis0 < isp0 < mfc in x, got {pos}"
    )


def test_y_coordinates_use_lane_y():
    """hw lane 노드의 y좌표가 LANE_Y['hw']와 일치한다."""
    from scenario_db.view.layout import LANE_Y
    view = _projection_to_view_response(SAMPLE_PROJECTION)
    for node in view.nodes:
        assert node.position["y"] == LANE_Y["hw"], (
            f"Node {node.data.id} y={node.position['y']}, expected {LANE_Y['hw']}"
        )


def test_edges_converted():
    """pipeline edges가 EdgeElement로 변환된다."""
    view = _projection_to_view_response(SAMPLE_PROJECTION)
    assert len(view.edges) == 2
    edge_types = {e.data.flow_type for e in view.edges}
    assert "OTF" in edge_types
    assert "M2M" in edge_types


def test_no_stub_positions():
    """position이 (0.0, 0.0) stub이 아니다."""
    view = _projection_to_view_response(SAMPLE_PROJECTION)
    stub_count = sum(1 for n in view.nodes if n.position["x"] == 0.0 and n.position["y"] == 0.0)
    assert stub_count == 0, f"{stub_count} nodes still have stub position (0,0)"


def test_category_to_lane_mapping():
    """CATEGORY_TO_LANE 상수가 올바른 값을 가진다."""
    assert CATEGORY_TO_LANE["camera"]  == "hw"
    assert CATEGORY_TO_LANE["codec"]   == "hw"
    assert CATEGORY_TO_LANE["display"] == "hw"
    assert CATEGORY_TO_LANE["memory"]  == "hw"


def test_isolated_node_gets_stage_0():
    """edge가 없는 isolated node는 stage_index 0 (x가 가장 작은 값)."""
    proj = {
        "scenario_id": "uc-test",
        "variant_id":  "v-test",
        "pipeline": {
            "nodes": [
                {"id": "llc", "ip_ref": "ip-llc-v2"},
                {"id": "csis0", "ip_ref": "ip-csis-v8"},
            ],
            "edges": [{"from": "csis0", "to": "csis0", "type": "OTF"}]
            # llc는 edge 없음 — isolated
        },
        "ip_catalog": [
            {"id": "ip-llc-v2",  "category": "memory"},
            {"id": "ip-csis-v8", "category": "camera"},
        ],
    }
    # isolated node가 있어도 오류 없이 ViewResponse를 반환해야 함
    # (edge self-loop 방지를 위해 valid edge로 수정)
    proj2 = {
        "scenario_id": "uc-test",
        "variant_id":  "v-test",
        "pipeline": {
            "nodes": [{"id": "llc", "ip_ref": "ip-llc-v2"}],
            "edges": [],
        },
        "ip_catalog": [{"id": "ip-llc-v2", "category": "memory"}],
    }
    view = _projection_to_view_response(proj2)
    assert len(view.nodes) == 1
    # isolated node는 stage 0에 배치됨 (x > LANE_LABEL_W)
    from scenario_db.view.layout import LANE_LABEL_W
    assert view.nodes[0].position["x"] > LANE_LABEL_W
