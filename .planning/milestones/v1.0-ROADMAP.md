## Overview

기존 하드코딩 샘플 뷰어를 DB-backed projection으로 전환하고,

## Milestone Summary

### Key Decisions

| Decision | Rationale |
|----------|-----------|
| Soft validation in ETL | 오류 수집 후 리포트 — DB 상태 유지, validate_loaded() |
| Resolver 결과 비영속 | runtime only, authored Review와 분리 — gate_executions 테이블 없음 |
| View API sample fallback 제거 | "Sample data trap" 방지 — demo 모드만 허용 |
| Simulation porting: Option B (선택적 포팅) | sim/ 패키지 신규 생성, ScenarioGraph 재구현 불필요 |
| DVFS 테이블 YAML화 | hw_config/dvfs-projectA.yaml — CSV 의존성 제거 |
| D-01 단일 정의 원칙 | PortBWResult/IPTimingResult는 evidence.simulation에서 re-import |
| monkeypatch 전략 (Phase 7-03) | run_simulation + load_runner_inputs_from_db 패치 — DVFS YAML 의존성 없이 통합 테스트 |
